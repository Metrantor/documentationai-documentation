# `public.build_full_codex_context_json(...)`

## Zweck

Baut den kompletten Beat-/Prompt-Codex-Kontext in einem Aufruf.

Er liefert:

- Standard-Codex

- Super-Codex

- Chapter-Codex

- World-Codex

jeweils als:

- ID-Arrays

- JSONB-Payloads

- gerenderte Prompt-Blöcke

## Signatur

```typescript
public.build_full_codex_context_json(
  p_beat_id uuid,
  p_text text,
  p_max_relation_depth integer default 2
) returns jsonb
```

## Parameter

| Parameter              | Typ       | Bedeutung                                                                |
| ---------------------- | --------- | ------------------------------------------------------------------------ |
| `p_beat_id`            | `uuid`    | Beat-Kontext                                                             |
| `p_text`               | `text`    | Freitext für Matching und Exclusions                                     |
| `p_max_relation_depth` | `integer` | Rekursionstiefe für `get_matching_codex_ids_from_text(...)`, Default `2` |

## Rückgabe

Ein großes JSONB-Objekt mit:

- `input`&#x20;

- `ids`&#x20;

- `payloads`&#x20;

- `rendered`&#x20;

## Wie die Funktion arbeitet

### Schritt 1

Lädt aus dem Beat:

- &#x20;Serie&#x20;

- &#x20;Kapitel&#x20;

- `effective_at`&#x20;

### Schritt 2

Baut die Standard-Basis über:

- `get_matching_codex_ids_from_text(...)`&#x20;

### Schritt 3

Filtert diese Basis zusätzlich auf:

- `is_active`&#x20;

- `valid_from / valid_to`&#x20;

### Schritt 4

Leitet daraus Super-Codex ab und filtert auch Super-Ziele auf:

- `is_active`&#x20;

- &#x20;Datum&#x20;

- &#x20;Exclusions&#x20;

### Schritt 5

Baut Chapter-Codex:

- &#x20;nur aktive Einträge&#x20;

- &#x20;zeitlich gültig&#x20;

- &#x20;nicht excluded&#x20;

### Schritt 6

Baut World-Codex:

- `always_include = true`&#x20;

- &#x20;aktiv&#x20;

- &#x20;zeitlich gültig&#x20;

- &#x20;nicht excluded&#x20;

### Schritt 7

Entfernt Super-Dubletten aus Standard/Chapter/World.

### Schritt 8

Erzeugt Payloads und Rendering.

## Aufrufvarianten

### Standard

```typescript
select public.build_full_codex_context_json(
  '18f58f5a-a454-4cdc-8a60-7eec80c1b007'::uuid,
  'Captain Sarvin stand auf der Brücke der Kalega. Der Reaktor machte Probleme.',
  2
);
```

### Ohne Relationen

```typescript
select public.build_full_codex_context_json(
  '18f58f5a-a454-4cdc-8a60-7eec80c1b007'::uuid,

  0

```

### Mit tieferer Relation

```typescript

```

## `CREATE OR REPLACE FUNCTION`

```typescript
create or replace function public.build_full_codex_context_json(
  p_beat_id uuid,
  p_text text,
  p_max_relation_depth integer default 2
)
returns jsonb
language plpgsql
stable
as $function$
declare
  v_series_text text;
  v_chapter_id uuid;
  v_effective_at timestamptz;
  v_result jsonb;
begin
  if p_max_relation_depth is null or p_max_relation_depth < 0 then
    p_max_relation_depth := 2;
  end if;

  select
    n.series_id::text,
    ch.id,
    s.starts_at
  into
    v_series_text,
    v_chapter_id,
    v_effective_at
  from public.beats b
  join public.scenes s
    on s.id = b.scene_id
  join public.chapters ch
    on ch.id = s.chapter_id
  join public.novels n
    on n.id = ch.novel_id
  where b.id = p_beat_id;

  if not found then
    raise exception 'Beat % wurde nicht gefunden', p_beat_id;
  end if;

  with params as (
    select
      coalesce(p_text, '') as raw_text,
      lower(coalesce(p_text, '')) as text_lower,
      v_effective_at as effective_at
  ),
  standard_base_raw as (
    select coalesce(
      public.get_matching_codex_ids_from_text(
        v_series_text,
        coalesce(p_text, ''),
        p_max_relation_depth
      ),
      '{}'::uuid[]
    ) as ids
  ),
  standard_base as (
    select coalesce(
      array(
        select distinct ce.id
        from unnest((select ids from standard_base_raw)) as u(id)
        join public.codex_entries ce
          on ce.id = u.id
        join params p
          on true
        where ce.is_active = true
          and (
            p.effective_at is null
            or (
              (ce.valid_from is null or ce.valid_from <= p.effective_at)
              and (ce.valid_to is null or ce.valid_to >= p.effective_at)
            )
          )
        order by ce.id
      ),
      '{}'::uuid[]
    ) as ids
  ),
  super_raw as (
    select coalesce(
      public.get_super_codex_ids((select ids from standard_base)),
      '{}'::uuid[]
    ) as ids
  ),
  super_filtered as (
    with super_input as (
      select distinct unnest((select ids from super_raw)) as codex_id
    )
    select coalesce(
      array(
        select si.codex_id
        from super_input si
        join public.codex_entries ce
          on ce.id = si.codex_id
        join params p
          on true
        where ce.is_active = true
          and (
            p.effective_at is null
            or (
              (ce.valid_from is null or ce.valid_from <= p.effective_at)
              and (ce.valid_to is null or ce.valid_to >= p.effective_at)
            )
          )
          and not exists (
            select 1
            from public.codex_exclusions ex
            where ex.entry_id = si.codex_id
              and p.text_lower like '%' || coalesce(ex.normalized_excluded_phrase, lower(ex.excluded_phrase)) || '%'
          )
        order by si.codex_id
      ),
      '{}'::uuid[]
    ) as ids
  ),
  standard_final as (
    select coalesce(
      public.remove_super_codex_duplicates(
        (select ids from standard_base),
        (select ids from super_filtered)
      ),
      '{}'::uuid[]
    ) as ids
  ),
  chapter_base as (
    select coalesce(
      array(
        select distinct ce.id
        from public.chapter_codex_entries cce
        join public.codex_entries ce
          on ce.id = cce.codex_entry_id
        join params p
          on true
        where cce.chapter_id = v_chapter_id
          and ce.is_active = true
          and (
            p.effective_at is null
            or (
              (ce.valid_from is null or ce.valid_from <= p.effective_at)
              and (ce.valid_to is null or ce.valid_to >= p.effective_at)
            )
          )
          and not exists (
            select 1
            from public.codex_exclusions ex
            where ex.entry_id = ce.id
              and p.text_lower like '%' || coalesce(ex.normalized_excluded_phrase, lower(ex.excluded_phrase)) || '%'
          )
        order by ce.id
      ),
      '{}'::uuid[]
    ) as ids
  ),
  chapter_final as (
    select coalesce(
      public.remove_super_codex_duplicates(
        (select ids from chapter_base),
        (select ids from super_filtered)
      ),
      '{}'::uuid[]
    ) as ids
  ),
  world_base as (
    select coalesce(
      array(
        select distinct ce.id
        from public.codex_entries ce
        join params p
          on true
        where ce.series_id = v_series_text
          and ce.is_active = true
          and ce.always_include = true
          and (
            p.effective_at is null
            or (
              (ce.valid_from is null or ce.valid_from <= p.effective_at)
              and (ce.valid_to is null or ce.valid_to >= p.effective_at)
            )
          )
          and not exists (
            select 1
            from public.codex_exclusions ex
            where ex.entry_id = ce.id
              and p.text_lower like '%' || coalesce(ex.normalized_excluded_phrase, lower(ex.excluded_phrase)) || '%'
          )
        order by ce.id
      ),
      '{}'::uuid[]
    ) as ids
  ),
  world_final as (
    select coalesce(
      public.remove_super_codex_duplicates(
        (select ids from world_base),
        (select ids from super_filtered)
      ),
      '{}'::uuid[]
    ) as ids
  ),
  standard_payload as (
    select coalesce(
      public.get_prompt_codex_payload_for_beat(
        (select ids from standard_final),
        p_beat_id
      ),
      '[]'::jsonb
    ) as payload
  ),
  super_payload as (
    select coalesce(
      public.get_prompt_codex_payload_for_beat(
        (select ids from super_filtered),
        p_beat_id
      ),
      '[]'::jsonb
    ) as payload
  ),
  chapter_payload as (
    select coalesce(
      public.get_prompt_codex_payload_for_beat(
        (select ids from chapter_final),
        p_beat_id
      ),
      '[]'::jsonb
    ) as payload
  ),
  world_payload as (
    select coalesce(
      public.get_prompt_codex_payload_for_beat(
        (select ids from world_final),
        p_beat_id
      ),
      '[]'::jsonb
    ) as payload
  )
  select jsonb_build_object(
    'input', jsonb_build_object(
      'series_id', v_series_text,
      'beat_id', p_beat_id,
      'chapter_id', v_chapter_id,
      'effective_at', v_effective_at,
      'text', coalesce(p_text, ''),
      'max_relation_depth', p_max_relation_depth
    ),
    'ids', jsonb_build_object(
      'standard_codex_ids', to_jsonb((select ids from standard_final)),
      'super_codex_ids', to_jsonb((select ids from super_filtered)),
      'chapter_codex_ids', to_jsonb((select ids from chapter_final)),
      'world_codex_ids', to_jsonb((select ids from world_final))
    ),
    'payloads', jsonb_build_object(
      'standard_codex', (select payload from standard_payload),
      'super_codex', (select payload from super_payload),
      'chapter_codex', (select payload from chapter_payload),
      'world_codex', (select payload from world_payload)
    ),
    'rendered', jsonb_build_object(
      'standard_codex', public.render_codex_payload((select payload from standard_payload), 'codex'),
      'super_codex', public.render_codex_payload((select payload from super_payload), 'super codex'),
      'chapter_codex', public.render_codex_payload((select payload from chapter_payload), 'chapter codex'),
      'world_codex', public.render_codex_payload((select payload from world_payload), 'world codex')
    )
  )
  into v_result;

  return v_result;
end;
$function$;
```

## Limitierungen und kritische Punkte

- &#x20;Die Standard-Basis hängt weiter von `get_matching_codex_ids_from_text(...)` ab.&#x20;

- &#x20;Wenn dort fachlich „zu viel“ oder „zu wenig“ gematcht wird, wirkt sich das hier auf alles aus.&#x20;

- `p_max_relation_depth` kann größere Kontexte und potenziell mehr Laufzeit erzeugen.&#x20;

- &#x20;Exclusions wirken hart: ein passender Eintrag fällt komplett raus.&#x20;

- &#x20;Die Funktion ist absichtlich mächtig, aber dadurch auch komplexer zu debuggen.
