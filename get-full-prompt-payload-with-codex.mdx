# `public.get_full_prompt_payload_with_codex(...)`

## Zweck

Diese Funktion ist die zentrale Komfortfunktion für den vollständigen Prompt-Kontext.

Sie kombiniert in **einem einzigen JSONB-Ergebnis**:

### A. Story-Kontext

aus:

- `public.get_prompt_payload_context(...)`

### B. Codex-Kontext

aus:

- `public.build_full_codex_context_json(...)`

Die Funktion liefert damit alles, was für den Promptbau typischerweise benötigt wird:

- History-Drafts

- History-Summaries

- lokaler Kapitelkontext

- Folgebeat-Kontext

- Standard-Codex

- Super-Codex

- Chapter-Codex

- World-Codex

Zusätzlich erzeugt sie einen flachen Bereich `prompt_blocks`, damit die Werte in n8n oder im Prompt-Script leicht ersetzt werden können.

---

## Signatur

```typescript
public.get_full_prompt_payload_with_codex(
  p_beat_id uuid,
  p_text text,
  p_compile_series boolean default false,
  p_required_only boolean default false,
  p_history_draft_limit integer default 12000,
  p_history_summary_limit integer default 6000,
  p_draft_before_limit integer default 12000,
  p_draft_after_limit integer default 12000,
  p_following_beat_limit integer default 6000,
  p_max_relation_depth integer default 2
) returns jsonb
```

---

## Parameter

| Parameter                 | Typ       | Bedeutung                                                                   |
| ------------------------- | --------- | --------------------------------------------------------------------------- |
| `p_beat_id`               | `uuid`    | Zielbeat, aus dem Kapitel- und Serienkontext abgeleitet wird                |
| `p_text`                  | `text`    | Freitext für das Codex-Matching                                             |
| `p_compile_series`        | `boolean` | Ob Story-History über mehrere Romane der Serie hinweg aufgebaut wird        |
| `p_required_only`         | `boolean` | Ob bei Inflows nur `required`-Inflows berücksichtigt werden                 |
| `p_history_draft_limit`   | `integer` | Maximale Länge für `history_drafts`                                         |
| `p_history_summary_limit` | `integer` | Maximale Länge für `history_summaries`                                      |
| `p_draft_before_limit`    | `integer` | Maximale Länge für `current_chapter_draft_before_beat`                      |
| `p_draft_after_limit`     | `integer` | Maximale Länge für `chapter_draft_after_beat`                               |
| `p_following_beat_limit`  | `integer` | Maximale Länge für `following_beat_texts`                                   |
| `p_max_relation_depth`    | `integer` | Maximale Rekursionstiefe für die Relationsauflösung im Codex-Basis-Matching |

---

## Rückgabe

Die Funktion gibt ein `jsonb` mit drei Hauptbereichen zurück:

### `prompt_context`

Technischer Story-Kontext:

- `history_drafts`&#x20;

- `history_summaries`&#x20;

- `current_chapter_draft_before_beat`&#x20;

- `chapter_draft_after_beat`&#x20;

- `following_beat_texts`&#x20;

- &#x20;Truncation-Flags&#x20;

### `codex_context`

Kompletter Codex-Output aus `build_full_codex_context_json(...)`:

- `input`&#x20;

- `ids`&#x20;

- `payloads`&#x20;

- `rendered`&#x20;

### `prompt_blocks`

Flache, direkt verwendbare Promptbausteine:

- &#x20;Story-Blöcke&#x20;

- &#x20;gerenderte Codex-Blöcke&#x20;

## SQL-Statement

```typescript
create or replace function public.get_full_prompt_payload_with_codex(
  p_beat_id uuid,
  p_text text,
  p_compile_series boolean default false,
  p_required_only boolean default false,
  p_history_draft_limit integer default 12000,
  p_history_summary_limit integer default 6000,
  p_draft_before_limit integer default 12000,
  p_draft_after_limit integer default 12000,
  p_following_beat_limit integer default 6000,
  p_max_relation_depth integer default 2
)
returns jsonb
language plpgsql
stable
as $function$
declare
  v_prompt_context record;
  v_codex_context jsonb;
begin
  if p_max_relation_depth is null or p_max_relation_depth < 0 then
    p_max_relation_depth := 2;
  end if;

  select *
  into v_prompt_context
  from public.get_prompt_payload_context(
    p_beat_id => p_beat_id,
    p_compile_series => p_compile_series,
    p_required_only => p_required_only,
    p_history_draft_limit => p_history_draft_limit,
    p_history_summary_limit => p_history_summary_limit,
    p_draft_before_limit => p_draft_before_limit,
    p_draft_after_limit => p_draft_after_limit,
    p_following_beat_limit => p_following_beat_limit
  );

  v_codex_context := public.build_full_codex_context_json(
    p_beat_id => p_beat_id,
    p_text => p_text,
    p_max_relation_depth => p_max_relation_depth
  );

  return jsonb_build_object(
    'prompt_context',
    jsonb_build_object(
      'history_drafts', v_prompt_context.history_drafts,
      'history_summaries', v_prompt_context.history_summaries,
      'current_chapter_draft_before_beat', v_prompt_context.current_chapter_draft_before_beat,
      'chapter_draft_after_beat', v_prompt_context.chapter_draft_after_beat,
      'following_beat_texts', v_prompt_context.following_beat_texts,
      'history_drafts_truncated', v_prompt_context.history_drafts_truncated,
      'history_summaries_truncated', v_prompt_context.history_summaries_truncated,
      'current_chapter_draft_before_beat_truncated', v_prompt_context.current_chapter_draft_before_beat_truncated,
      'chapter_draft_after_beat_truncated', v_prompt_context.chapter_draft_after_beat_truncated,
      'following_beat_texts_truncated', v_prompt_context.following_beat_texts_truncated
    ),

    'codex_context',
    coalesce(v_codex_context, '{}'::jsonb),

    'prompt_blocks',
    jsonb_build_object(
      'history_drafts', v_prompt_context.history_drafts,
      'history_summaries', v_prompt_context.history_summaries,
      'current_chapter_draft_before_beat', v_prompt_context.current_chapter_draft_before_beat,
      'chapter_draft_after_beat', v_prompt_context.chapter_draft_after_beat,
      'following_beat_texts', v_prompt_context.following_beat_texts,
      'codex', coalesce(v_codex_context -> 'rendered' ->> 'standard_codex', ''),
      'super_codex', coalesce(v_codex_context -> 'rendered' ->> 'super_codex', ''),
      'chapter_codex', coalesce(v_codex_context -> 'rendered' ->> 'chapter_codex', ''),
      'world_codex', coalesce(v_codex_context -> 'rendered' ->> 'world_codex', '')
    )
  );
end;
$function$;
```
