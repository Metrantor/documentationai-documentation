# `public.get_matching_codex_details_for_beat(...)`

## Zweck

Ermittelt alle **gültigen Codex-Detail-Einträge** für eine Menge von Haupteinträgen im Kontext eines Beats.

Diese Funktion ist jetzt die zentrale Stelle für:

- Datumsprüfung auf Haupteintrags-Ebene

- Datumsprüfung auf Detail-Ebene

- Exclusions

- rekursive Relationen

- POV

- Variablenbedingungen

- Text-Trigger / Regex-Trigger

## Signatur

```typescript
public.get_matching_codex_details_for_beat(
  p_codex_entry_ids uuid[],
  p_beat_id uuid
)
returns table(...)
```

## Parameter

| Parameter           | Typ      | Bedeutung                                        |
| ------------------- | -------- | ------------------------------------------------ |
| `p_codex_entry_ids` | `uuid[]` | Startmenge von Haupteinträgen                    |
| `p_beat_id`         | `uuid`   | Beat-Kontext für Zeit, POV, Text und Story-State |

## Rückgabe

Die Funktion liefert eine Tabellenmenge mit:

| Feld                        | Typ           | Bedeutung                |
| --------------------------- | ------------- | ------------------------ |
| `conditional_entry_id`      | `uuid`        | ID des Detail-Eintrags   |
| `codex_entry_id`            | `uuid`        | zugehöriger Haupteintrag |
| `title`                     | `text`        | Titel des Details        |
| `codex_text`                | `text`        | eigentlicher Detailtext  |
| `note`                      | `text`        | optionale Notiz          |
| `sort_order`                | `integer`     | Sortierung               |
| `condition_mode`            | `text`        | `all` oder `any`         |
| `requires_pov_character_id` | `uuid`        | optionaler POV-Filter    |
| `valid_from`                | `timestamptz` | gültig ab                |
| `valid_to`                  | `timestamptz` | gültig bis               |

## Wie die Funktion arbeitet

### Schritt 1: Beat-Kontext laden

Sie lädt aus dem Beat:

- &#x20;Serie&#x20;

- &#x20;POV&#x20;

- `effective_at`&#x20;

- &#x20;normalisierten Beat-Text&#x20;

### Schritt 2: Seed-Haupteinträge filtern

Aus den übergebenen IDs bleiben nur solche übrig, die:

- `is_active = true`&#x20;

- &#x20;zeitlich gültig sind:&#x20;

  - `valid_from` leer oder \<= `effective_at`&#x20;

  - `valid_to` leer oder >= `effective_at`&#x20;

### Schritt 3: Seed-Exclusions anwenden

Wenn eine Exclusion-Phrase des Haupteintrags im Beat-Text vorkommt, fliegt der Eintrag raus.

### Schritt 4: Rekursive Relationen verfolgen

Es werden relationell verknüpfte Haupteinträge gezogen, aber nur wenn sie:

- &#x20;aktive Relation haben&#x20;

- `pull_into_context = true`&#x20;

- &#x20;selbst aktiv sind&#x20;

- &#x20;zeitlich gültig sind&#x20;

- &#x20;nicht durch Exclusion gesperrt werden&#x20;

### Schritt 5: Candidate-Details laden

Für die so entstandenen Haupteinträge werden passende `codex_conditional_entries` geladen, wenn:

- &#x20;der Detail-Eintrag aktiv ist&#x20;

- &#x20;der POV passt&#x20;

- &#x20;das Detail-Datum passt&#x20;

### Schritt 6: Variablenbedingungen prüfen

Über `evaluate_story_variable_condition(...)`.

### Schritt 7: Text-Trigger prüfen

Pro Detail wird geprüft:

- &#x20;wie viele aktive Trigger existieren&#x20;

- &#x20;wie viele davon matchen&#x20;

`phrase`:

- `like '%...%'`&#x20;

`regex`:

- `~*`&#x20;

### Schritt 8: Endgültige Filterung

Ein Detail ist gültig, wenn:

- &#x20;keine Bedingungen existieren\
  &#x20;oder die Bedingungen erfüllt sind&#x20;

- &#x20;und:&#x20;

  - &#x20;keine aktiven Trigger existieren\
    &#x20;oder mindestens ein Trigger matcht&#x20;

## `CREATE OR REPLACE FUNCTION`

```typescript
create or replace function public.get_matching_codex_details_for_beat(
  p_codex_entry_ids uuid[],
  p_beat_id uuid
)
returns table(
  conditional_entry_id uuid,
  codex_entry_id uuid,
  title text,
  codex_text text,
  note text,
  sort_order integer,
  condition_mode text,
  requires_pov_character_id uuid,
  valid_from timestamptz,
  valid_to timestamptz
)
language sql
stable
as $function$
with target as (
  select
    sr.id as series_id,
    b.id as beat_id,
    ch.pov_character_entry_id as pov_character_entry_id,
    s.starts_at as effective_at,
    lower(coalesce(nullif(b.beat_text, ''), nullif(b.draft_text, ''), '')) as beat_text_normalized
  from public.beats b
  join public.scenes s
    on s.id = b.scene_id
  join public.chapters ch
    on ch.id = s.chapter_id
  join public.novels n
    on n.id = ch.novel_id
  join public.series sr
    on sr.id = n.series_id
  where b.id = p_beat_id
),
input_ids as (
  select distinct unnest(coalesce(p_codex_entry_ids, '{}'::uuid[])) as codex_entry_id
),
seed_entries as (
  select ce.id as codex_entry_id
  from input_ids i
  join public.codex_entries ce
    on ce.id = i.codex_entry_id
  join target t
    on true
  where ce.is_active = true
    and (
      t.effective_at is null
      or (
        (ce.valid_from is null or ce.valid_from <= t.effective_at)
        and (ce.valid_to is null or ce.valid_to >= t.effective_at)
      )
    )
),
seed_excluded as (
  select distinct se.codex_entry_id
  from seed_entries se
  join public.codex_exclusions ex
    on ex.entry_id = se.codex_entry_id
  join target t
    on true
  where t.beat_text_normalized like '%' || coalesce(ex.normalized_excluded_phrase, lower(ex.excluded_phrase)) || '%'
),
seed_allowed as (
  select se.codex_entry_id
  from seed_entries se
  left join seed_excluded sx
    on sx.codex_entry_id = se.codex_entry_id
  where sx.codex_entry_id is null
),
relation_tree as (
  with recursive rel as (
    select
      sa.codex_entry_id as root_entry_id,
      sa.codex_entry_id as codex_entry_id,
      array[sa.codex_entry_id] as path
    from seed_allowed sa

    union all

    select
      rel.root_entry_id,
      cr.target_entry_id as codex_entry_id,
      rel.path || cr.target_entry_id
    from rel
    join public.codex_relations cr
      on cr.source_entry_id = rel.codex_entry_id
     and cr.is_active = true
     and cr.pull_into_context = true
    join public.codex_entries ce
      on ce.id = cr.target_entry_id
     and ce.is_active = true
    join target t
      on true
    where not (cr.target_entry_id = any(rel.path))
      and (
        t.effective_at is null
        or (
          (ce.valid_from is null or ce.valid_from <= t.effective_at)
          and (ce.valid_to is null or ce.valid_to >= t.effective_at)
        )
      )
      and not exists (
        select 1
        from public.codex_exclusions ex
        where ex.entry_id = cr.target_entry_id
          and t.beat_text_normalized like '%' || coalesce(ex.normalized_excluded_phrase, lower(ex.excluded_phrase)) || '%'
      )
  )
  select distinct
    rel.root_entry_id,
    rel.codex_entry_id
  from rel
),
candidate_codex_ids as (
  select distinct codex_entry_id
  from relation_tree
),
current_story_state as (
  select
    gss.variable_id,
    gss.variable_key,
    gss.variable_label,
    gss.value_json
  from public.get_story_state_at_beat(p_beat_id, 'gross') gss
),
candidate_entries as (
  select
    cce.id,
    cce.codex_entry_id,
    cce.title,
    cce.codex_text,
    cce.note,
    cce.sort_order,
    cce.condition_mode,
    cce.requires_pov_character_id,
    cce.valid_from,
    cce.valid_to,
    cce.series_id
  from public.codex_conditional_entries cce
  join candidate_codex_ids fci
    on fci.codex_entry_id = cce.codex_entry_id
  join public.codex_entries ce
    on ce.id = cce.codex_entry_id
  join target t
    on t.series_id = cce.series_id
  where cce.is_active = true
    and ce.is_active = true
    and (
      cce.requires_pov_character_id is null
      or cce.requires_pov_character_id = t.pov_character_entry_id
    )
    and (
      t.effective_at is null
      or (
        (cce.valid_from is null or cce.valid_from <= t.effective_at)
        and (cce.valid_to is null or cce.valid_to >= t.effective_at)
      )
    )
),
conditions_joined as (
  select
    ce.id as conditional_entry_id,
    coalesce(ce.condition_mode, 'all') as condition_mode,
    cond.id as condition_id,
    cond.sort_order as condition_sort_order,
    sv.id as variable_id,
    sv.type as variable_type,
    css.value_json as current_value_json,
    cond.operator,
    cond.compare_value_json,
    coalesce(
      public.evaluate_story_variable_condition(
        sv.type,
        css.value_json,
        cond.operator,
        cond.compare_value_json
      ),
      false
    ) as condition_matches
  from candidate_entries ce
  left join public.codex_conditional_entry_variable_conditions cond
    on cond.conditional_entry_id = ce.id
  left join public.story_variables sv
    on sv.id = cond.variable_id
  left join current_story_state css
    on css.variable_id = cond.variable_id
),
condition_summary as (
  select
    cj.conditional_entry_id,
    max(cj.condition_mode) as condition_mode,
    count(cj.condition_id) as total_conditions,
    coalesce(
      bool_and(cj.condition_matches) filter (where cj.condition_id is not null),
      true
    ) as all_conditions_match,
    coalesce(
      bool_or(cj.condition_matches) filter (where cj.condition_id is not null),
      false
    ) as any_condition_match
  from conditions_joined cj
  group by cj.conditional_entry_id
),
text_trigger_matches as (
  select
    ce.id as conditional_entry_id,
    count(tt.id) as total_triggers,
    count(*) filter (
      where tt.match_mode = 'phrase'
        and t.beat_text_normalized like '%' || lower(tt.trigger_value) || '%'
    ) as matching_phrase_triggers,
    count(*) filter (
      where tt.match_mode = 'regex'
        and t.beat_text_normalized ~* tt.trigger_value
    ) as matching_regex_triggers
  from candidate_entries ce
  join target t
    on true
  left join public.codex_conditional_entry_text_triggers tt
    on tt.conditional_entry_id = ce.id
   and tt.is_active = true
  group by ce.id
),
text_trigger_summary as (
  select
    tm.conditional_entry_id,
    coalesce(tm.total_triggers, 0) as total_triggers,
    (
      coalesce(tm.matching_phrase_triggers, 0)
      + coalesce(tm.matching_regex_triggers, 0)
    ) as matching_triggers
  from text_trigger_matches tm
),
valid_entries as (
  select
    ce.*
  from candidate_entries ce
  left join condition_summary cs
    on cs.conditional_entry_id = ce.id
  left join text_trigger_summary ts
    on ts.conditional_entry_id = ce.id
  where
    (
      coalesce(cs.total_conditions, 0) = 0
      or (
        coalesce(cs.condition_mode, 'all') = 'all'
        and cs.all_conditions_match = true
      )
      or (
        coalesce(cs.condition_mode, 'all') = 'any'
        and cs.any_condition_match = true
      )
    )
    and (
      coalesce(ts.total_triggers, 0) = 0
      or coalesce(ts.matching_triggers, 0) > 0
    )
)
select
  ve.id as conditional_entry_id,
  ve.codex_entry_id,
  ve.title,
  ve.codex_text,
  ve.note,
  ve.sort_order,
  ve.condition_mode,
  ve.requires_pov_character_id,
  ve.valid_from,
  ve.valid_to
from valid_entries ve
order by ve.codex_entry_id, ve.sort_order, ve.title;
$function$;
```

## Limitierungen und kritische Punkte

- `phrase` ist einfacher Teilstring-Match, keine Wortgrenzen.&#x20;

- &#x20;Regex kann stark sein, aber auch zu breit oder zu eng matchen.&#x20;

- &#x20;Die Funktion kann bei vielen Relationen + vielen Bedingungen + vielen Triggern schwerer werden.&#x20;

- &#x20;Ein ungültiger Regex kann Laufzeitfehler auslösen.&#x20;

- &#x20;Exclusions greifen auf Haupteintrags-Ebene, nicht als feinere Trigger-Logik.&#x20;

- &#x20;Aktuell gibt es keine Gruppenlogik wie „alle Trigger müssen matchen“.
