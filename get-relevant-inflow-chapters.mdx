---
title: get_relevant_inflow_chapters
---
##

## Was die Funktion macht

Diese Funktion findet die **einmündenden Kapitelstränge** zu einem Zielkapitel.

Einfach gesagt:

> Sie schaut auf den Mainflow und fragt:\
> „Welche Seitenwege laufen in diese Hauptkapitel hinein?“

Diese Seitenwege heißen bei dir **Inflows**.

---

## Wofür die Funktion gut ist

Du brauchst sie, wenn du nicht nur den normalen Hauptweg der Geschichte betrachten willst, sondern auch die zusätzlichen Kapitel, die in diesen Hauptweg hineinmünden.

Das ist wichtig für:

- Nexus-Kapitel

- alternative Handlungsstränge

- kontextreiche KI-Prompts

- strukturierte Kontextsammlung

---

## Welche Parameter sie hat

### `p_chapter_id uuid`

Das Zielkapitel, von dem aus gedacht wird.

Die Funktion fragt also:

> Welche inflows sind für dieses Kapitel und seine relevanten Vorgänger wichtig?

---

### `p_compile_series boolean default false`

Steuert, wie weit zurück geschaut wird.

#### Wenn `false`

Es wird nur innerhalb des aktuellen Romans zurückverfolgt.

#### Wenn `true`

Es wird auch über Vorgängerromane hinweg zurückverfolgt.

---

### `p_required_only boolean default false`

Steuert, ob nur verpflichtende inflows berücksichtigt werden.

#### Wenn `false`

Alle inflows werden einbezogen.

#### Wenn `true`

Nur inflows mit `is_required = true` werden genommen.

---

## Was die Funktion zurückgibt

Die Funktion gibt **eine Tabelle mit mehreren Zeilen** zurück.

Jede Zeile beschreibt **einen inflow**.

### Rückgabefelder

#### `owner_chapter_id uuid`

Das Hauptkapitel, zu dem dieser inflow gehört.

#### `inflow_chapter_id uuid`

Das Kapitel des einmündenden Seitenwegs.

#### `inflow_sort_order integer`

Die Reihenfolge dieses inflows.

#### `is_required boolean`

Ob dieser inflow als verpflichtend markiert ist.

#### `owner_depth integer`

Wie weit das Hauptkapitel vom Ziel entfernt ist.

- `0` = Zielkapitel

- `1` = direkter Vorgänger des Zielkapitels

- `2` = davor

- usw.

---

## Wie die Funktion denkt

Sie arbeitet in zwei Schritten:

### Schritt 1

Sie bestimmt zuerst alle relevanten Kapitel des Mainflows.

### Schritt 2

Zu jedem dieser Kapitel schaut sie in `chapter_inflows`, ob es inflows gibt.

Dann sortiert sie diese inflows nach:

- Tiefe im Mainflow

- inflow `sort_order`

## Einfaches Aufrufbeispiel

```typescript
select *
from public.get_relevant_inflow_chapters(
  p_chapter_id := '11111111-1111-1111-1111-111111111111',
  p_compile_series := false,
  p_required_only := false
);
```

### Bedeutung

> „Zeig mir alle inflows für dieses Kapitel und seine Mainflow-Vorgänger im aktuellen Roman.“

---

## Beispiel nur für verpflichtende inflows

```typescript
select *
from public.get_relevant_inflow_chapters(
  p_chapter_id := '11111111-1111-1111-1111-111111111111',
  p_compile_series := true,
  p_required_only := true
);
```

### Bedeutung

> „Zeig mir nur die wichtigen inflows, auch über Vorgängerromane hinweg.“

---

## Beispiel einer Rückgabe

```typescript
owner_chapter_id                        inflow_chapter_id                       inflow_sort_order  is_required  owner_depth
------------------------------------    ------------------------------------    -----------------  -----------  -----------
aaaa-...                                x111-...                               1                  true         0
aaaa-...                                x222-...                               2                  false        0
bbbb-...                                y111-...                               1                  true         1
```

### Das bedeutet

- &#x20;Zum Zielkapitel (`owner_depth = 0`) gibt es zwei inflows&#x20;

- &#x20;Zum vorherigen Hauptkapitel (`owner_depth = 1`) gibt es noch einen inflow

# SQL

```typescript
create or replace function public.get_relevant_inflow_chapters(
  p_chapter_id uuid,
  p_compile_series boolean default false,
  p_required_only boolean default false
)
returns table (
  owner_chapter_id uuid,
  inflow_chapter_id uuid,
  inflow_sort_order integer,
  is_required boolean,
  owner_depth integer
)
language sql
stable
as $$
  with mainflow as (
    select *
    from public.get_relevant_mainflow_chapters(p_chapter_id, p_compile_series)
  )
  select
    mf.chapter_id as owner_chapter_id,
    ci.inflow_chapter_id,
    ci.sort_order as inflow_sort_order,
    ci.is_required,
    mf.depth as owner_depth
  from mainflow mf
  join public.chapter_inflows ci
    on ci.chapter_id = mf.chapter_id
  where
    (not p_required_only or ci.is_required = true)
  order by
    mf.depth desc,
    ci.sort_order,
    ci.inflow_chapter_id;
$$;
```
